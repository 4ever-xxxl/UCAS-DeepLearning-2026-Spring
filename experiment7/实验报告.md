# 实验7 神经网络语言模型实验报告

## 一、概述

本实验实现了一个基于 LSTM 的词级神经网络语言模型。任务是在 Penn Treebank (PTB) 语料库上，根据历史词序列预测下一个词，并用困惑度（Perplexity, PPL）评估模型对测试语料的建模能力。实验指导书要求在 PTB 上训练和评估语言模型，测试集 PPL 低于 80。

本方案使用 PyTorch 实现经典 PTB 两层 LSTM 结构：词嵌入层将 10,000 个词映射为 1500 维向量，随后经过 2 层 LSTM 建模上下文依赖，最后通过线性层输出下一个词的概率分布。训练采用截断 BPTT、SGD、全局梯度裁剪和学习率衰减，并启用 AMP、TF32 和 `torch.compile()` 提升 RTX 5070 上的训练效率。

## 二、实验环境

- 操作系统：Linux
- Python：3.13
- 深度学习框架：PyTorch 2.11.0+cu130
- 训练硬件：NVIDIA GeForce RTX 5070（12GB）
- CPU：Intel i5-14600KF（6P+8E, 20 线程）
- 数据集：Penn Treebank (PTB) simple-examples

## 三、解决方案

### 3.1 网络结构设计

模型为词级语言模型，输入为形状 `[batch_size, num_steps]` 的词 ID 序列，输出为每个时间步的下一个词分布。整体结构为：

`Embedding(10000, 1500) -> Dropout(0.65) -> LSTM(1500, 1500, num_layers=2, dropout=0.65) -> Dropout(0.65) -> Linear(1500, 10000)`

其中：

- Embedding 层学习词的稠密表示。
- 两层 LSTM 负责捕获词序列中的短程和长程依赖。
- Dropout 用于抑制 PTB 小语料上的过拟合。
- Linear 层将 LSTM 隐状态投影到 10,000 维词表空间。
- softmax 在交叉熵损失内部完成。

模型参数量：66,034,000。

核心代码如下：

```python
class PTBLanguageModel(nn.Module):
    def __init__(self, vocab_size, embedding_dim, hidden_dim,
                 num_layers, dropout, init_scale, tied_weights):
        super().__init__()
        self.features = nn.ModuleDict({
            "embedding": nn.Embedding(vocab_size, embedding_dim),
            "lstm": nn.LSTM(
                input_size=embedding_dim,
                hidden_size=hidden_dim,
                num_layers=num_layers,
                dropout=dropout if num_layers > 1 else 0.0,
                batch_first=True,
            ),
        })
        self.dropout = nn.Dropout(dropout)
        self.classifier = nn.Sequential(nn.Linear(hidden_dim, vocab_size))
```

### 3.2 损失函数设计

语言模型的目标是最大化真实下一个词的条件概率。本实验使用交叉熵损失：

```python
criterion = nn.CrossEntropyLoss()
logits, hidden = model(inputs, hidden)
loss = criterion(logits, targets.reshape(-1))
```

若平均交叉熵为 `loss`，则困惑度为：

```python
perplexity = exp(loss)
```

PPL 越低，表示模型给真实文本分配的概率越高。与分类任务不同，本实验不以准确率为核心指标，但脚本仍统计 next-word top-1 accuracy 作为辅助观察指标。

### 3.3 优化器设计

优化器采用 PTB 语言模型常用的 SGD：

```python
optimizer = torch.optim.SGD(
    model.parameters(),
    lr=config.learning_rate,
    weight_decay=config.weight_decay,
)
torch.nn.utils.clip_grad_norm_(model.parameters(), config.max_grad_norm)
```

默认 `large` preset 的关键超参数如下：

| 超参数 | 数值 |
|---|---:|
| Batch size | 20 |
| BPTT 长度 (`num_steps`) | 35 |
| Epochs | 55 |
| Embedding dim | 1500 |
| Hidden dim | 1500 |
| LSTM layers | 2 |
| Dropout | 0.65 |
| 初始学习率 | 1.0 |
| 梯度裁剪 | 10.0 |
| 学习率衰减 | 第 15 轮起每轮乘以 1/1.15 |

硬件加速策略：

- CUDA 可用时启用 AMP 混合精度。
- 启用 TF32：`torch.set_float32_matmul_precision("high")`。
- 启用 `torch.compile(model, mode="reduce-overhead")`。
- DataLoader 使用 PTB 原始顺序，不 shuffle，以保持 stateful BPTT 的上下文连续性。

### 3.4 创新点

1. **PyTorch 复现 TensorFlow 指导书流程**：指导书示例基于 TensorFlow，本实验将 PTB 数据生产、LSTM 建图、BPTT 和 PPL 评估完整迁移到 PyTorch。
2. **可复用 preset 机制**：提供 `test/small/medium/large` 四组配置，既能快速 smoke test，也能用 large preset 冲击 PPL<80。
3. **鲁棒数据下载**：优先下载指导书给出的 Mikolov 压缩包，失败时回退到 PTB 原始三文件镜像，并用 MD5 校验。
4. **现代训练优化**：在保持经典 LSTM 架构的前提下，加入 AMP、TF32 和 `torch.compile()`。
5. **结果可视化与样例分析**：提供 loss/PPL 曲线、next-word top-5 示例和文本续写样例，便于报告展示。

## 四、实验分析

### 4.1 数据集介绍

PTB simple-examples 是语言模型常用基准数据集。实验使用预处理后的词级文本：

- 训练集：约 93 万 token
- 验证集：约 7.4 万 token
- 测试集：约 8.2 万 token
- 词表大小：10,000
- 特殊标记：`<eos>` 表示句末，`<unk>` 表示低频或未知词。

数据处理流程：

1. 将换行替换为 `<eos>`，保留句子边界。
2. 只用训练集统计词频并构建词表。
3. 将 train/valid/test 文本映射为词 ID。
4. 按 `[batch_size, batch_len]` reshape 成连续流。
5. 每个 batch 取长度为 `num_steps` 的输入序列，目标序列整体右移一位。

结果分析：

1. **训练初期困惑度下降快**：前若干轮模型从接近随机预测逐步学习到高频词、标点和 `<eos>` 模式，验证 PPL 会快速下降。
2. **Dropout 对 PTB 很关键**：PTB 训练集规模不大，大隐层 LSTM 参数多，较强 dropout 能抑制过拟合。
3. **梯度裁剪稳定 BPTT**：循环网络在长序列反传中容易出现梯度爆炸，global norm clipping 使 SGD 训练稳定。
4. **学习率衰减决定后期表现**：large preset 前 14 轮保持较大学习率，之后逐轮衰减，通常能在后期显著改善验证/测试 PPL。
5. **测试集达到要求**：最终测试 PPL 为 78.94，低于指导书要求的 80；验证 PPL 为 82.61，说明模型在后期已接近泛化瓶颈。
6. **next-word accuracy 不是主要指标**：词表有 10,000 类，且很多上下文存在多个合理下一个词，因此 PPL 更能反映语言模型质量。

## 五、总结

本实验完成了 PTB 词级 LSTM 神经网络语言模型的完整实现，包括数据下载与校验、词表构建、BPTT 批生成、两层 LSTM 建模、训练验证测试、checkpoint 保存和结果可视化。默认 `large` preset 对齐经典 PTB LSTM 设置，最终测试 PPL 为 78.94，达到实验指导书要求的 PPL<80。

后续可改进方向：

1. 引入 weight tying，减少参数并可能提升泛化。
2. 使用 variational dropout 或 AWD-LSTM 的正则化策略进一步降低 PPL。
3. 替换为 Transformer 语言模型，对比循环网络和自注意力结构在 PTB 上的表现。
4. 增加 top-k/top-p 解码，用于更稳定的文本续写展示。
