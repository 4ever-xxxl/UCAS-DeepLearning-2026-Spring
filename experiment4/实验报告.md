# 实验四 基于 Transformer 的神经机器翻译实验报告

## 一、概述

- **任务**：神经机器翻译（Neural Machine Translation, NMT）——把中文句子翻译为英文。
- **数据集**：NiuTrans 中英平行语料（TM-training-set 100,000 句对，Test-set 1,000 句中文及对应英文参考）。
- **解决方案概要**：完整从零实现 "Attention Is All You Need" 中的 Transformer Encoder–Decoder 架构；使用 SentencePiece BPE 子词分词解决 OOV 问题；训练-验证-测试三段式流水线，采用 teacher-forcing 训练 + label smoothing + warmup-rsqrt 学习率调度，最终用 beam search 解码评估。

## 二、实验环境

- 操作系统：Linux（WSL2，Ubuntu 24）
- Python：3.13
- 深度学习框架：PyTorch 2.11.0+cu130、SentencePiece 0.2.1、sacrebleu 2.6.0
- 训练硬件：NVIDIA GeForce RTX 5070（12 GB VRAM, Blackwell）、Intel i5-14600KF
- 数据集：NiuTrans 中英平行语料

## 三、解决方案

### 3.1 数据预处理与分词

原始 NiuTrans 语料是以空格切分的"伪 token"序列，但**词级别词表会带来高 OOV**：训练集中文唯一词 5.4 万、英文 3.5 万，即使把词表扩到 5 万，开发集仍有 ~46% 中文 token 被映射为 `<unk>`，BLEU 始终为 0。
本实验改用 **SentencePiece BPE** 子词分词，源语言（zh）与目标语言（en）各训练一个独立 BPE 模型，词表大小均为 16,000。子词覆盖了所有训练语料字符（character_coverage=0.9995/1.0），OOV 率几乎为 0。

特殊 token ID 与 BPE 模型对齐：

| Token | ID |
|-------|----|
| `<pad>` | 0 |
| `<unk>` | 1 |
| `<s>` (BOS) | 2 |
| `</s>` (EOS) | 3 |

数据切分：

| 切分 | 数量 | 来源 |
|------|------|------|
| 训练集 | 99,000 句对 | TM-training-set 随机打乱后取前 99k |
| 验证集 | 1,000 句对 | TM-training-set 随机打乱后取后 1k |
| 测试集 | 1,000 句对 | Niu.test.txt（zh）+ Niu.test.reference 解析所得（en） |

### 3.2 网络结构设计

模型 `TransformerNMT` 严格按照 "Attention Is All You Need" 实现，使用 **Pre-LayerNorm** 变体以提升训练稳定性：

```
src_ids → src_embed × √d_model → +PE → [EncoderLayer × 6] → enc_norm
                                                    ↓
tgt_ids → tgt_embed × √d_model → +PE → [DecoderLayer × 6] → dec_norm → output_proj (tied with tgt_embed)
```

每个 EncoderLayer 由「Self-Attention + FFN」两个残差块组成；每个 DecoderLayer 由「Masked Self-Attention + Cross-Attention + FFN」三个残差块组成：

```python
class DecoderLayer(nn.Module):
    def forward(self, x, enc_out, tgt_mask, src_mask):
        n = self.norm1(x); x = x + self.dropout(self.self_attn(n, n, n, tgt_mask))
        n = self.norm2(x); x = x + self.dropout(self.cross_attn(n, enc_out, enc_out, src_mask))
        x = x + self.dropout(self.ff(self.norm3(x)))
        return x
```

注意力使用 PyTorch 内置的 `F.scaled_dot_product_attention`（启用 Flash Attention），mask 采用其 bool 约定 **True = attend**：

```python
def src_attend_mask(src):                      # encoder/cross-attn key mask
    return (src != PAD_IDX).unsqueeze(1).unsqueeze(2)

def tgt_attend_mask(tgt):                      # decoder self-attn: pad ∧ causal
    pad_ok  = (tgt != PAD_IDX).unsqueeze(1).unsqueeze(2)
    causal  = torch.tril(torch.ones(T, T, device=tgt.device, dtype=torch.bool))
    return pad_ok & causal
```

> 注：调试期间发现 `F.scaled_dot_product_attention` 与 `nn.MultiheadAttention` 的 bool mask 语义相反，前者 True=attend、后者 True=mask。最初的实现误用了后者的语义，导致 decoder 在 teacher forcing 时反而能"看到未来"，训练 loss 正常下降但自回归推理 BLEU=0。修正 mask 语义后训练立刻收敛。

模型超参：

| 超参 | 值 |
|------|----|
| `d_model` | 256 |
| `n_heads` | 8 |
| Encoder layers | 6 |
| Decoder layers | 6 |
| `d_ff` | 1024 |
| Dropout | 0.1 |
| 词嵌入与输出投影权重共享（weight tying） | ✓ |

总参数量：**19,252,224**（19.25 M）。

### 3.3 损失函数设计

使用 **带 label smoothing 的交叉熵**，平滑系数 ε=0.1，忽略 PAD 位置：

```python
criterion = nn.CrossEntropyLoss(ignore_index=PAD_IDX, label_smoothing=0.1)
```

label smoothing 把硬 one-hot 目标 `y` 软化为 `(1−ε)·y + ε/V`，可有效抑制模型对训练分布的过度拟合，提升 BLEU 指标（Vaswani et al. 2017 的经验）。

### 3.4 优化器与学习率调度

使用 **AdamW**（β₁=0.9，β₂=0.98，ε=1e-9，weight_decay=1e-5），峰值学习率 7e-4。

学习率调度采用 Transformer 论文的 **warmup-rsqrt** 公式：

```python
def lr_lambda(step):
    s = step + 1
    if s < warmup_steps:
        return s / warmup_steps             # 线性 warmup
    return math.sqrt(warmup_steps / s)      # 之后按 1/√step 衰减
```

`warmup_steps=2000`（约前 2.6 个 epoch），梯度裁剪 `clip_grad_norm_=1.0`，AMP（bfloat16）混合精度。

### 3.5 解码策略

- **训练 / 验证**：teacher forcing；验证用 greedy 解码计算 BLEU 用于早停选模。
- **最终测试**：同时给出 greedy 和 **beam search（beam=5，length penalty=0.6）** 的 BLEU。

beam search 加入长度惩罚 `((5+L+1)/(5+1))^α` 避免偏好过短句。

## 四、实验分析

### 4.1 数据集介绍

NiuTrans 是东北大学自然语言处理实验室开源的中英平行语料样本数据。原始包包含 199,630 句对的训练集，本实验使用其中 100,000 句对的子集。

数据特点：
- 中文已用 ICTCLAS2011 分词，词与词之间空格分隔。
- 英文已 tokenize、lower-case，标点亦被分离为独立 token。
- 句子平均长度：中文 26 token、英文 30 token。

数据预处理流水线：

1. 修复原始数据流水线（旧版本 `dev.zh / dev.en` 行错位、`test.en` 内容误为中文）：通过 `_parse_paired_blocks` 正确解析 NiuTrans 引用文件 `zh ⏎ blank ⏎ en` 的三行块格式。
2. 训练集 / 验证集从 100k TM 数据中随机切分（seed=0，dev_size=1000）。
3. 各自训练 SentencePiece BPE 模型（zh：`character_coverage=0.9995`；en：`character_coverage=1.0`）。
4. 编码后每条目标序列前后追加 `<s>` 和 `</s>`，`<pad>` 用于 batch 内动态填充。

**翻译样例**（来自 `outputs/test_samples.json`）：

| # | 翻译输出 | 参考译文 |
|---|----------|----------|
| 1（极佳） | second , comprehensive management . | second , comprehensive management . |
| 2（极佳） | tunisia highly praised china 's foreign and domestic policies pursued by chinese leaders and expressed his deep admiration for china 's achievements in economic and social development . | tunisia highly appreciated the domestic and foreign policies that the chinese leaders have pursued and deeply admired china 's achievements in economic and social development . |
| 3（良好） | at the beginning of the new millennium , people of all walks of life took advantage of this gathering to open up new ideas and draw up a new blueprint for world development in the new century . | at the beginning of the new millennium , the participants took advantage of this gathering to explore new ideas and chart a new course for world development in the new century . |
| 4（一般） | article 20 the people 's bank of china set up a renminbi issuance facility and set up branches in its branch branch branches to ensure the issuance of the renminbi . | article 20 the people 's bank of china sets up the renminbi issuance treasury and branch treasuries in its branch institutions to be in charge of the safekeeping of the renminbi issuance funds . |
| 5（错误） | shi guangsheng expressed his thanks for this . | shi guangsheng thanked him for his support . |

**结果分析（4 个要点）**：

1. **修复 mask 语义后训练立刻收敛**：原始实现误把 padding/causal mask 直接作为 `F.scaled_dot_product_attention` 的 attend mask 传入，导致 decoder 训练时反向地「只能看见 pad/未来」。修复后从 epoch 1 BLEU=1.0 → epoch 4 BLEU=16.7，目标值在 4 个 epoch 内即被超越。
2. **BPE 子词分词是机器翻译的必要条件**：词级别词表在 100k 中英语料下产生 ~38–51% 的 `<unk>` 率，BLEU 几乎为 0。BPE 把 vocab 限定在 16k 同时让 OOV 率近乎为 0。
3. **beam search 比 greedy 提升约 1 BLEU**：beam=5 + length penalty=0.6 给出 27.79（greedy 为 26.86），符合标准 NMT 经验。
4. **20 epoch 后收益递减**：epoch 20 时 BLEU 已达 25.3，到 epoch 30 才升至 26.5。考虑到 19M 参数 + 100k 句对 这一规模，后续若想进一步突破，需要更大的模型或额外的回译数据（back-translation）。

错误模式观察：

- **重复 token**："branch branch branches"。可在解码加入 no-repeat-ngram 启发式缓解。
- **专有名词漏译/错译**：如样例 13 把姓名误处理。这与训练数据中长尾命名实体不足有关。
- **长难句信息丢失**：参考译文较长时模型有时只翻译前半部分。

## 五、总结

- 实验完成情况：✅ 完成完整 Transformer 实现、数据流水线、训练 / 评估代码、结果可视化、报告与 PPT 提纲。
- 是否达到预期目标：✅ Test BLEU 26.86（greedy）/ 27.79（beam=5），**远高于**实验目标 BLEU ≥ 14。
- 可改进方向：
  1. **更大模型**：升至 d_model=512、heads=8、6/6 层（Transformer-base，~65M）通常可到 ~30 BLEU。
  2. **数据增广**：使用回译（back-translation）扩充训练数据。
  3. **更精细解码**：no-repeat-ngram、coverage penalty、ensemble。
  4. **预训练初始化**：使用 mBART / NLLB 等预训练模型微调。
  5. **多参考评估**：当前测试集每句仅 1 参考，多参考 BLEU 通常更宽容。
