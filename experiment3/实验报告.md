# 实验三 自动写诗实验报告

## 一、概述

本实验实现了一个基于 LSTM 循环神经网络的唐诗自动生成模型。任务为字符级语言建模：给定一首唐诗的首句（如"湖光秋月两相和"），模型自动续写后续诗句，生成符合汉语语法和唐诗格律的完整诗篇。

数据集为课程提供的预处理唐诗数据集（tang.npz），包含 57,580 首唐诗，词汇量 8,293 个汉字。实验采用 3 层 LSTM + LayerNorm + Dropout 的架构（与指导书示例的单层 LSTM 不同），训练后模型在测试集上达到困惑度 67.85，能够生成具有唐诗风格的续写诗句。

## 二、实验环境

- 操作系统：Linux (WSL2)
- Python：3.13.12
- 深度学习框架：PyTorch 2.11.0+cu130、torchvision 0.26.0+cu130
- 训练硬件：NVIDIA GeForce RTX 5070（12GB GDDR7, Blackwell 架构）
- CPU：Intel i5-14600KF（6P+8E, 20 线程）
- 数据集：tang.npz（唐诗数据集）

## 三、解决方案

### 3.1 网络结构设计

模型采用 3 层单向 LSTM 作为核心特征提取器，输入为字符索引序列，输出为下一个字符的概率分布。架构如下：

`Embedding(8293, 384) → LSTM(384, 768, num_layers=3, dropout=0.3) → LayerNorm(768) → Dropout(0.3) → Linear(768, 8293)`

Embedding 层将离散汉字索引映射为 384 维稠密向量；3 层 LSTM 逐层提取字符间的短程和长程依赖关系，层间 Dropout=0.3 防止过拟合；LayerNorm 归一化 LSTM 输出以稳定深层网络训练；最终通过 Linear 层投影回 8,293 维词表空间，经 softmax 得到下一个字的概率分布。

模型参数量：22,557,925。

```python
class PoetryModel(nn.Module):
    def __init__(self, vocab_size, embedding_dim, hidden_dim, num_layers, lstm_dropout):
        super().__init__()
        self.hidden_dim = hidden_dim
        self.num_layers = num_layers
        self.features = nn.ModuleDict({
            "embedding": nn.Embedding(vocab_size, embedding_dim),
            "lstm": nn.LSTM(embedding_dim, hidden_dim,
                            num_layers=num_layers, batch_first=True,
                            dropout=lstm_dropout if num_layers > 1 else 0.0),
        })
        self.classifier = nn.Sequential(
            nn.LayerNorm(hidden_dim),
            nn.Dropout(lstm_dropout),
            nn.Linear(hidden_dim, vocab_size),
        )
```

### 3.2 损失函数设计

采用交叉熵损失函数 `nn.CrossEntropyLoss(ignore_index=8292)`，其中 `ignore_index=8292` 对应填充符 `</s>` 的索引，确保模型仅在有效字符上计算损失，不被大量填充符干扰训练。

训练时采用 Teacher Forcing 策略：每个时间步的输入为真实的前一个字符，而非模型自身生成的字符。输入为序列前 124 个 token（`seq[:, :-1]`），目标为序列后 124 个 token（`seq[:, 1:]`）。

```python
criterion = nn.CrossEntropyLoss(ignore_index=pad_idx)  # pad_idx=8292
# ...
inputs = seq[:, :-1]
targets = seq[:, 1:].reshape(-1)
logits, _ = model(inputs)
loss = criterion(logits, targets)
```

### 3.3 优化器设计

采用 Adam 优化器，初始学习率 5e-4，weight_decay=1e-4。配合 `ReduceLROnPlateau` 学习率调度器（mode='min', factor=0.5, patience=3），当验证损失连续 3 轮未下降时将学习率减半。

正则化策略包括：LSTM 层间 Dropout=0.3、LayerNorm 归一化、梯度裁剪（max_norm=1.0）。

为充分利用 RTX 5070 硬件，启用 TF32 精度（`torch.set_float32_matmul_precision('high')`）、AMP 混合精度训练（`GradScaler + autocast`）、以及 `torch.compile(model, mode="reduce-overhead")`。

```python
optimizer = torch.optim.Adam(model.parameters(), lr=5e-4, weight_decay=1e-4)
scheduler = torch.optim.lr_scheduler.ReduceLROnPlateau(
    optimizer, mode="min", factor=0.5, patience=3)
scaler = GradScaler(device=device.type, enabled=use_amp)
```

### 3.4 创新点

与实验指导书示例模型的差异：
1. **3 层 LSTM** 替代单层 LSTM：更好建模唐诗的层级结构（字→词→句→联→篇）
2. **引入 LayerNorm**：归一化每层 LSTM 输出，稳定深层网络训练，加速收敛
3. **更大的嵌入和隐层维度**：embedding_dim=384（vs 指导书未指定，通常 128-256）、hidden_dim=768（vs 通常 256-512）
4. **完整的正则化体系**：LSTM 层间 Dropout + 输出 Dropout + LayerNorm + 梯度裁剪
5. **现代训练优化**：TF32 精度 + torch.compile + AMP 混合精度，充分利用 Blackwell 架构 Tensor Core

## 四、实验分析

### 4.1 数据集介绍

- **来源**：课程网站提供的预处理唐诗数据集 tang.npz
- **规模**：57,580 首唐诗
- **词汇量**：8,293 个汉字（含特殊符号）
- **序列格式**：每首诗固定 125 字符，右对齐，左侧 `</s>` 填充，格式为 `</s>*N <START> 诗句 <EOP>`
- **特殊符号**：`<START>`（idx=8291, 句首）、`<EOP>`（idx=8290, 句末）、`</s>`（idx=8292, 填充）
- **数据划分**：训练 46,064（80%）/ 验证 5,758（10%）/ 测试 5,758（10%）

数据预处理：将诗词中的汉字转为词表索引（int64），无需额外分词（中文以字为单位）。训练时 `CrossEntropyLoss(ignore_index=8292)` 自动忽略填充位置的损失。

**结果分析：**

1. **收敛过程**：模型在前 15 轮快速收敛，困惑度从 380 迅速降至 75；第 15-28 轮进入瓶颈期，困惑度在 72-76 之间震荡；第 29 轮学习率衰减（5e-4→2.5e-4）后突破瓶颈，最终验证困惑度降至 68.5。这表明 ReduceLROnPlateau 调度策略有效帮助模型跳出局部最优。

2. **生成质量**：生成的诗歌在词汇选择上呈现明显的唐诗风格（山色、风雨、松窗、竹影、金銮等意象），韵律和句法基本符合五言/七言格式。"白日依山尽"的续写尤为出色——"青山古寺深。山僧多古寺，僧院有闲吟"营造了与原诗一致的山水禅意。

3. **退化问题**：部分生成结果存在重复现象（如连续多句以相同字/词开头），这是 LSTM 语言模型的常见退化问题。"床前明月光"的续写出现了"妾心不可见"的无限循环。可通过温度采样或 top-k/top-p 采样缓解。

4. **长程依赖**：125 字符的序列长度对 LSTM 的长程依赖建模提出了较高要求。模型在诗歌后半部分偶尔出现语义断裂（如从山水突然切换到闺怨主题）。引入注意力机制或增大隐藏层维度可能进一步改善。

5. **硬件利用**：RTX 5070 的 12GB 显存完全满足训练需求（峰值显存约 1.5GB），TF32 + torch.compile + AMP 三重优化使每轮训练仅需约 25 秒，30 轮总计约 12 分钟。

## 五、总结

本实验完成了基于 PyTorch 的唐诗自动生成任务，实现了从数据加载、模型构建、训练到生成测试的完整流程。模型采用 3 层 LSTM + LayerNorm + Dropout 的架构设计（区别于指导书示例的单层 LSTM），在 57,580 首唐诗上训练后，测试集困惑度为 67.85。给定任意首句，模型能够生成符合唐诗基本格律和语言习惯的续写诗句，达到了实验要求。

后续改进方向：（1）引入 Attention 机制或改用 Transformer/GPT 架构以增强长程依赖建模；（2）实现温度采样和 top-k/top-p 解码以缓解重复生成问题；（3）引入诗词韵律约束（平仄、押韵）以进一步提升诗歌质量；（4）扩展为藏头诗等更复杂的生成任务。
