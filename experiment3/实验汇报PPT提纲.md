## 第 3 页：实验任务概述

- **任务定义**：字符级唐诗语言模型，给定首句自动续写后续诗句
- **应用场景**：NLG 文本生成、AI 辅助创作、古典文学数字化
- **核心挑战**：
  - 唐诗格律严格（五言/七言、平仄、押韵）
  - 长程语义一致性（全诗主题统一）
  - 避免生成退化（重复、无意义循环）
- **方法**：基于 3 层 LSTM 的字符级神经语言模型

## 第 4 页：数据集介绍

- **来源**：课程网站预处理唐诗数据集 tang.npz
- **规模**：57,580 首唐诗
- **词汇量**：8,293 个汉字（含特殊符号）
- **序列格式**：固定 125 字符，右对齐，左侧 `</s>` 填充
  - 格式：`</s>*N <START> 诗句 <EOP>`
- **特殊符号**：
  - `<START>`（8291）：句首标记
  - `<EOP>`（8290）：句末标记
  - `</s>`（8292）：填充符号（训练时忽略）
- **数据划分**：训练 46,064 / 验证 5,758 / 测试 5,758

## 第 5 页：模型结构设计

- **Embedding**（8293 → 384）：汉字索引 → 稠密语义向量
- **3 层 LSTM**（384 → 768, dropout=0.3）：
  - Layer 1：学习字符级共现模式
  - Layer 2：学习词语和短语结构
  - Layer 3：学习句子和篇章级连贯性
- **LayerNorm**（768）：归一化 LSTM 输出，稳定训练
- **Dropout**（0.3）：防止过拟合
- **Linear**（768 → 8293）：投影到词表空间
- **总参数量**：22,557,925

## 第 6 页：损失函数与优化器设计

- **损失函数**：`CrossEntropyLoss(ignore_index=8292)`
  - 忽略填充符 `</s>`，仅在有效字符上计算损失
  - 训练策略：Teacher Forcing（输入真实前文而非预测值）
- **优化器**：Adam（lr=5e-4, weight_decay=1e-4）
- **学习率调度**：ReduceLROnPlateau（factor=0.5, patience=3）
  - 验证损失停滞 3 轮 → 学习率减半
- **正则化**：
  - LSTM 层间 Dropout=0.3
  - 输出层 Dropout=0.3
  - LayerNorm 归一化
  - 梯度裁剪（max_norm=1.0）
- **硬件加速**：TF32 精度 + AMP 混合精度 + torch.compile

## 第 7 页：创新点说明

与指导书示例模型的差异：

| 方面 | 指导书示例 | 本方案 |
|------|-----------|--------|
| LSTM 层数 | 1 层 | **3 层** |
| 归一化 | 无 | **LayerNorm** |
| 嵌入维度 | 未指定（通常 128-256） | **384** |
| 隐层维度 | 未指定（通常 256-512） | **768** |
| 正则化 | 仅 LSTM 自带 | **Dropout + LayerNorm + GradClip** |
| 训练加速 | 无 | **TF32 + AMP + torch.compile** |

核心思路：更深的 LSTM 更好地建模唐诗层级结构（字→词→句→联→篇），LayerNorm 和 Dropout 协同防止深层网络的过拟合和不稳定。

## 第 8 页：训练过程与配置

| 超参数 | 值 |
|--------|-----|
| Batch size | 128 |
| Epochs | 30 |
| Learning rate | 5e-4 |
| Weight decay | 1e-4 |
| Embedding dim | 384 |
| Hidden dim | 768 |
| LSTM layers | 3 |
| LSTM dropout | 0.3 |
| Gradient clipping | 1.0 |
| 硬件 | RTX 5070 (12GB) |
| 训练加速 | TF32 + AMP + torch.compile |
| 训练耗时 | ~12 分钟（30 epochs） |

关键策略：ReduceLROnPlateau 调度器在瓶颈期自动降低学习率，在第 29 轮触发突破。

## 第 9 页：实验结果

- **最佳验证困惑度**：68.50（Epoch 30）
- **测试困惑度**：67.85
- **训练趋势**：PPL 380（Epoch 1）→ 75（Epoch 15，瓶颈）→ 68（Epoch 29-30，调度突破）

**生成示例**：

| 输入首句 | 模型续写 |
|---------|---------|
| 湖光秋月两相和 | 风雨萧萧日日斜。山色不知人不见，山川不是客无家… |
| 朝辞白帝彩云间 | 白云飞去不可攀。青山白云不可见，白云飞鸟飞飞还… |
| 白日依山尽 | 青山古寺深。山僧多古寺，僧院有闲吟。石壁松声远… |

## 第 10 页：结果分析与讨论

1. **收敛分析**：前 15 轮快速下降（PPL 380→75），中间 13 轮瓶颈震荡，学习率衰减后突破至 68.5。验证了调度策略的有效性。
2. **生成质量**：词汇呈现明显唐诗风格（山水、风月、宫苑意象），格律符合五言/七言格式。白日依山尽续写质量最佳。
3. **退化问题**：部分样本出现重复循环（LSTM 典型问题），可通过温度采样/top-k/top-p 缓解。
4. **长程依赖**：125 字符序列对 LSTM 有一定挑战，诗歌后半部分偶有主题偏离。
5. **硬件效率**：12GB 显存仅用 ~1.5GB，TF32+compile+AMP 三重优化下 30 轮仅需 ~12 分钟。

## 第 11 页：总结与改进方向

- **完成情况**：实现了基于 3 层 LSTM 的唐诗自动生成，测试困惑度 67.85，模型能够根据任意首句生成符合唐诗格律的续写诗句。网络设计有自己的方案（与指导书不同），达到了实验要求。
- **改进方向**：
  1. 引入 Attention 或 Transformer/GPT 架构增强长程依赖
  2. 温度采样 + top-k/top-p 解码缓解重复生成
  3. 加入韵律约束（平仄、押韵）提升诗歌质量
  4. 扩展为藏头诗等复杂生成任务
